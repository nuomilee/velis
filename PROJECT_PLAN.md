# 项目规划

## 1. 项目目标

实现一个基于 Python 3.10 + PySide6 的桌面整理工具，核心能力包括：

- 文件重命名
- 文件分类存放
- 桌面盒子展示与操作

## 2. 产品定位

面向 Windows 桌面场景，帮助用户自动整理桌面文件，并通过可视化盒子快速访问整理后的内容。

## 3. 功能拆解

### 3.1 文件重命名
- 输入：桌面文件
- 处理逻辑：
  - 识别文件扩展名与类型
  - 清理原始文件名中的噪声字符
  - 按统一规则生成新名称
- 输出：统一风格命名后的文件名

命名规则：
- `{分类前缀}_{清洗后的原名}_{YYYYMMDD_HHMMSS}{扩展名}`
- 例：`DOC_项目方案_20260323_121500.docx`

### 3.2 文件分类存放
- 基于扩展名初步判断
- 基于文件名关键词进行语义补充判断
- 对 `.lnk` 快捷方式优先使用名称关键词分类

分类目录：
- 文档
- 图片
- 视频
- 音频
- 压缩包
- 程序/快捷方式
- 表格数据
- 演示文稿
- 设计素材
- 代码开发
- 其他

### 3.3 桌面盒子
- 每个分类一个盒子
- 半透明浅灰背景
- 圆角边框
- 可拖动
- 可缩放
- 网格吸附
- 列表显示文件
- 双击打开文件
- 右键菜单支持打开所在目录、刷新

## 4. 技术设计

- 网格吸附

- 刷新盒子

#### `app.py`
- 应用入口装配

## 5. 核心流程

1. 扫描桌面文件
2. 跳过系统目录和工具自身工作目录
3. 对每个文件执行：
   - 提取扩展名
   - 基于扩展名分类
   - 若为 `.lnk` 或分类不明确，结合文件名关键词再分类
   - 生成统一新文件名
   - 移动到对应分类目录
4. 整理完成后刷新桌面盒子
5. 在盒子中展示分类结果并支持打开

## 7. 当前版本边界

- 当前 `.lnk` 仅依据文件名进行推断，不解析快捷方式目标路径
- 当前“AI整理”采用规则引擎方式模拟语义判断
- 自动对齐网格为本地坐标吸附，不包含多显示器高级布局保存
